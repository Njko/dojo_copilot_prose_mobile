Industrialiser GitHub Copilot en Entreprise

Le Problème

🤔

Pourquoi Copilot fonctionne bien pour les projets perso
mais galère sur les codebases d'entreprise ?

La réalité en entreprise

Copilot ne connaît pas les règles de votre équipe :

  • Ignore les patterns et conventions spécifiques à l'équipe
  • Utilise des librairies interdites (violations sécurité/conformité)
  • Génère du code techniquement correct mais contextuellement faux
  • Produit des résultats incohérents selon les développeurs

Le problème du "Vibe Coding"

Prompts improvisés :

  • Demander et espérer le meilleur
  • Ré-expliquer le contexte à chaque fois
  • Qualité incohérente
  • Connaissances piégées dans l'historique

Résultat :

  • Marche pour les tâches simples
  • Échoue à l'échelle
  • Impossible d'imposer des standards
  • Pas de reproductibilité

La Solution

💡

AI Native Development

Traiter les directives IA comme des artefacts d'ingénierie
avec la même rigueur que le code

Principes du développement AI Native

  • Codifier le contexte — Les standards vivent dans des fichiers versionnés
  • Automatiser l'application — Le contexte s'applique automatiquement via des patterns
  • Standardiser les workflows — Commandes réutilisables pour les tâches courantes
  • Spécialiser les agents — Personas IA dédiés avec des responsabilités claires
  • Packager pour l'échelle — Partager les capacités entre équipes

Les principes PROSE

Une approche systématique pour des workflows IA fiables

PROSE : Cinq piliers

P rogressive Disclosure — Utilisation de contexte efficacement

R educed Scope — Complexité générable

O rchestrated Composition — Flexibité, réutilisabilité

S afety Boundaries — Reliability, security, verifiability

E xplicit Hierarchy — Modularité, héritage

PROSE : Mémotechnique

P rompts — Commandes réutilisables invoquées explicitement

R eliability — Résultats cohérents et prévisibles

O rchestration — Workflows multi-agents avec handoffs

S kills — Capacités packagées découvertes par intention

E ngineering — Standards d'équipe auto-appliqués

PROSE n'est pas

  • Un framework — PROSE est un "style" comme REST
  • Un format de fichier — Les primitives sont une implémentation du style
  • Spécifique à un modèle — LLM/Agent Agnostic
  • Spécifique à un outil — VSCode, XCode, ... les contraintes sont universelles

Ce que PROSE Résoud

  • Le "context overload" — Avec le dépassement des limites de la fenêtre de contexte, les modèles perdent le fil de la discussion
  • Le déficit d'accompagnement — Des demandes vagues induisent des réponses vagues. Comment rendre le résultat prévisible? Répétable?
  • La fin du prompt engineering? — On arrête avec les interactions en one shot, et on pense workflows

Instructions

.instructions.md

Standards d'équipe auto-appliqués via patterns glob

Comment fonctionnent les Instructions

Point clé : Les instructions sont automatiquement incluses dans le contexte de Copilot quand vous travaillez sur des fichiers correspondants.

---
applyTo: "**/*.py"
---
# Standards Python

- Utiliser structlog pour tous les logs (jamais print ou logging stdlib)
- Toutes les fonctions nécessitent des docstrings Google-style
- Utiliser Pydantic pour la validation des données
- Suivre les patterns existants dans la codebase

Avantages des Instructions

  • Zéro friction — Pas de contexte manuel nécessaire
  • Cohérence — Mêmes standards pour tous
  • Versionné — Les standards évoluent avec le code
  • Scopé — Règles différentes selon les types de fichiers

Comment fonctionnent les Prompts

---
mode: agent
description: Génère une documentation complète pour les modules Python
---
# Générateur de Documentation

## Votre tâche
Analyser le fichier courant et générer la documentation en suivant nos
[standards de documentation](../instructions/documentation-standards.instructions.md).

## Étapes
1. Lire le fichier courant entièrement
2. Identifier toutes les fonctions, classes et méthodes publiques
3. Générer des docstrings Google-style pour chacune
4. Ajouter la documentation au niveau module
5. Inclure des exemples d'utilisation si approprié

Avantages des Prompts

  • Reproductibilité — Même workflow à chaque fois
  • Découvrabilité — Tapez / pour voir les commandes disponibles
  • Composabilité — Référencer des fichiers d'instructions
  • Partageabilité — Accès pour toute l'équipe via git

Agents

.agent.md

Personas IA spécialisés avec handoffs

Pattern d'architecture Agent

Agent Analyseur

  • Accès lecture seule
  • Analyse approfondie du code
  • Utilise les outils search/usages
  • Produit un rapport structuré

Agent Générateur

  • Accès écriture
  • Création de fichiers ciblée
  • Suit le rapport d'analyse
  • Responsabilité unique

Le bouton Handoff permet un transfert fluide entre agents

Pourquoi séparer les Agents ?

  • Séparation des responsabilités — Analyse vs. génération
  • Sécurité — Limiter l'accès écriture à l'agent spécialisé
  • Qualité — Meilleurs résultats avec des tâches ciblées
  • Auditabilité — Points de contrôle clairs pour la revue

# Skill Générateur de Documentation

## Description
Génère une documentation complète suivant les standards d'équipe.

## Quand l'utiliser
- L'utilisateur demande de "documenter ce code"
- L'utilisateur veut "ajouter des docstrings"
- L'utilisateur demande une "documentation API"

## Fichiers inclus
- instructions/documentation-standards.instructions.md
- prompts/generate-docs.prompt.md
- agents/doc-analyzer.agent.md
- agents/doc-writer.agent.md

Les Skills permettent

  • Découverte par intention — L'agent trouve le bon skill automatiquement
  • Partage inter-équipes — Packager et distribuer via APM / Skills.sh / etc.
  • Versioning — Les skills évoluent indépendamment
  • Composition — Construire sur les skills existants

Les Skills contiennent

  • Un fichier racine — SKILL.md qui décrit le coeur
  • Des instructions — Des compléments de règles de bonne pratique à appliquer au chargement du skill
  • Des prompts — Les tâches répétables que peut utiliser le skill
  • Des références — exemples que l'agent peut charger dans le contexte si besoin
  • Des scripts — des scripts exécutables pour automatiser certaines tâches

Leçons clés

🎓

Leçon 1 : Le contexte est roi

La qualité des résultats IA est directement proportionnelle à la qualité du contexte fourni.

  • Ne comptez pas sur les prompts improvisés
  • Codifiez vos standards dans des fichiers versionnés
  • Laissez l'automatisation gérer l'injection de contexte

Leçon 2 : Séparation des responsabilités

Les agents spécialisés surpassent les prompts généralistes.

  • Les analyseurs doivent analyser (lecture seule)
  • Les générateurs doivent générer (écritures ciblées)
  • Les handoffs permettent des points de contrôle qualité

Leçon 3 : Construire pour l'échelle

Ce qui marche pour un développeur doit marcher pour toute l'équipe.

  • Packager les workflows en Skills
  • Distribuer via APM CLI
  • Activer la découverte par intention

Leçon 4 : Accepter les contraintes

La réalité entreprise inclut du code legacy, des règles de conformité et des frontières d'équipe.

  • Documenter ce qui ne peut pas changer (NE PAS MODIFIER)
  • Encoder les exigences de conformité dans les instructions
  • Travailler avec les patterns existants, pas contre eux

Anti-patterns à éviter

⚠️

Ce qu'il ne faut pas faire

  • Vibe coding — Prompts improvisés sans structure
  • Dump de contexte — Coller des fichiers entiers dans le chat
  • Agents fourre-tout — Agents génériques pour tout
  • Ignorer les patterns existants — Laisser l'IA inventer de nouvelles conventions
  • Sauter l'analyse — Générer sans comprendre
